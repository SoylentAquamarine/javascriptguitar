Data Pipeline — Javascript Guitar

Blueprint · Last updated 2026-05-02

No Server-Side Data Pipeline

Javascript Guitar is a fully static site. There is no server, no API, no database, and no data transmission of any kind. All interaction happens entirely within the browser using preloaded assets.

The only "data" in this project is GIF images on disk. They are loaded once by the browser and swapped in-place on user interaction.

Browser-Side Interaction Flow

The closest equivalent to a data pipeline is the image-swap cycle that powers the fretboard visualizer:

  • Page load: all GIF variants are preloaded into browser cache via Image() objects.
  • User clicks or hovers a fret/string position.
  • An inline event handler computes the target GIF filename from the position.
  • document.images[n].src is replaced with the preloaded GIF path.
  • The browser renders the swapped image — no network request is made.

No Applicable Pipeline Features

The following pipeline concepts do not apply to this project:

  • API calls — none
  • Request routing — none
  • Authentication — none
  • Data transformation — none
  • Caching strategy — browser default only
  • Error handling — none beyond browser-native image load failures

Workflows — Javascript Guitar

Blueprint · Last updated 2026-05-02

No Programmatic Workflow System

Javascript Guitar does not have an application workflow engine. There are no task queues, state machines, or multi-step automated processes. All interaction is direct and synchronous: user action → immediate visual response.

The "workflow" in this project is a guitarist exploring the fretboard — it is entirely in the user's hands, not the application's.

User-Facing Tools

The site provides several independent tools, each on its own HTML page. These are not linked programmatically — the user navigates between them manually.

Development Workflow

When modifying the project, the only applicable workflow is:

  • Edit files in optimized/ only — never touch original/.
  • Open the HTML file directly in a browser to test (no dev server needed).
  • Commit to main — GitHub Pages deploys automatically on push.

Architecture — Javascript Guitar

Blueprint · Last updated 2026-05-02

Overview

Javascript Guitar is a zero-dependency static site. There is no build step, no framework, no server, and no runtime other than the browser. The architecture is two self-contained directories served directly from GitHub Pages.

Original Version — Core Pattern

The 1990s code uses a preload-and-swap technique that was standard before CSS and modern DOM APIs existed:

  1. On page load, Image() objects are created for every GIF variant and loaded into browser cache.
  2. Inline onmouseover / onclick handlers call a swap function.
  3. The swap function replaces document.images[n].src with the target GIF path.
  4. Because all images are preloaded, every swap is instantaneous with no network request.

GIF Naming Convention

Filenames encode musical position. Examples:

  • a_note21.gif — A string, note 21
  • note215.gif — Note 21, string 5
  • barre3.gif — Barre chord at fret 3

The naming scheme is implicit in the original code — there is no lookup table or manifest. Both versions rely on the same conventions and the same image files in optimized/images/.

Constraints

  • The original/ directory is permanently frozen — no changes, ever.
  • All new development happens in optimized/.
  • No build tools, bundlers, or package managers are used or needed.
  • No external CDN dependencies — the site must work fully offline once loaded.
